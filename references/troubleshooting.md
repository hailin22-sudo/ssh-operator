# 远程部署问题排查速查

## 连接不上 SSH

1. 确认 IP、端口、用户名正确。
2. 确认云服务商安全组已放行 SSH 端口（默认 22）。
3. 如果使用密码，确认未启用仅密钥登录。
4. 如果使用密钥，确认私钥权限为 `600`。

## 文件上传失败

1. 确认 `REMOTE_ROOT` 目录存在且有写权限。
2. 路径分隔符使用 `/`，即使在 Windows 本地。
3. 大文件夹建议使用 `rsync` 或 `tar + scp`。

## Docker 服务起不来

1. 查看日志：`docker-compose logs --tail=100 <service>`。
2. 检查端口占用：`netstat -tlnp` 或 `ss -tlnp`。
3. 检查 `.env` 文件是否正确上传到服务器。
4. 检查 volume 挂载路径是否存在。

## 页面 502 / 无法访问

1. 容器是否真的在运行：`docker-compose ps`。
2. Nginx 反代配置是否正确。
3. 宝塔防火墙和云安全组是否放行对应端口。
4. 域名 DNS 是否解析到服务器 IP。

## 数据库 / SQLite 报错

1. 确认 volume 持久化目录存在且容器有权限写入。
2. 不要把数据库文件放在会被覆盖的代码目录里。
3. 备份后再执行可能破坏数据的操作。

## 部署后代码没生效

1. 确认上传到了正确的远程路径。
2. 确认容器已重建（`docker-compose up -d --build`）。
3. 确认浏览器/CDN 没有缓存旧页面。
4. 进入容器检查文件：`docker exec <container> cat <path>`。
